## 海外访问中国大陆，到底卡在哪？

如果你是海外华人或留学生，应该遇到过这些场景：

- 🎬 在加拿大想看 **B站**，缓冲三十秒；看 **爱奇艺** 弹"该地区不可用"
- 🎵 用 **网易云音乐** 听华语歌，提示"由于版权方要求……"
- 🎮 玩 **王者荣耀国服 / 原神 / LOL 国服**，延迟动辄 400ms 起步，团战全靠走位预判
- 💬 用 **微信支付 / 支付宝** 给国内亲友转账失败，被风控拦截
- 🛒 上 **淘宝 / 京东**，加载半天打不开，账号被强制短信验证

这些问题的根源有三类：

| 问题 | 表现 | 根因 |
|---|---|---|
| **地理延迟** | TCP 握手 300ms+ | 物理距离远，跨太平洋海底光缆 |
| **地区版权** | HTTP 403 / 451，"该地区不可用" | 服务端基于 IP 地理位置主动拦截 |
| **DNS 污染** | DNS 解析失败或返回错误 IP | 跨境 DNS 路径被污染或运营商劫持 |

而**国内 ISP 对海外流量并不友好**：除非有专门的海外接入加速，否则海外用户的体验只会越来越差。

## 为什么不用 ping？

很多人测网络习惯用 `ping`，但 `ping` 走的是 **ICMP 协议**，而：

- 国内大量服务在边缘节点**丢弃 ICMP** 包 → 你 ping 不通但 HTTPS 正常
- ICMP 的延迟和真实业务的 TCP / HTTPS 延迟**经常对不上**
- `ping` 不会告诉你 DNS 是否被污染、HTTP 是否被拦截

**这个工具用 TCP 三次握手 + HTTP HEAD 做检测**，更接近你的浏览器和 App 实际用的协议。
